- [Bugfix] Pin the build-time `setuptools` used inside the isolated PEP 517 build environments that uv/pip create when building Python packages from source. Previously the latest `setuptools` was fetched to build every sdist regardless of the version installed in the venv, which made builds non-reproducible and broke them when `setuptools` 81 removed `pkg_resources` (e.g. when building `loremipsum`, a transitive dependency of `edx-ora2`). The constraint is set via `UV_BUILD_CONSTRAINT`/`PIP_CONSTRAINT` in the base image stage, so it applies to all requirement installs (base.txt, assets.txt, development.txt, extra requirements and editable installs). (by @abdul.muqadim)
